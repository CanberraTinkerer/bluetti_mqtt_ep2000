The Sequence:

2. Why you are getting Exception 3
Beyond the missing write to 2027, there are two common reasons for Exception 3 in this specific range:

Read Count (Block Size): Register 3500 is a "struct" type. In the Bluetti V2 protocol, the device often limits the number of registers you can read in a single request to 25 or 32. If you are attempting to read the entire 3500-3599 range (100 registers) in one packet, the device will reject it with Exception 3.
Encrypted Wrapper (P0x17): If you are using a standard Modbus tool to send a raw 0x03 (Read Holding Registers) command, the EP2000 firmware will often return Exception 3. This is because these registers are part of the "V2 Encrypted" space. The app uses Function Code 0x17 (wrapped in AES-CBC encryption) to access these. Raw Modbus access is often disabled for historical/statistical data to protect data integrity.

Confirm Protocol: Ensure you are using the encrypted packet format. If the device sees a plain Modbus header for a "Protected" register range, it will return an error.
